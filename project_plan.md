# Project Plan

## Aim
The aim of this project is to create a practical Software Quality Handbook for a small software company. The handbook will explain the best practices the company should follow to improve consistency, reduce defects, and make it easier for engineers to work in a more reliable way.

## Topics
The handbook will cover the following topics:
- Task Estimation in Scrum
- Code Reviews
- Testing Strategy

## Main Tasks
The main tasks for this project are:
- set up the GitHub repository
- create the project plan
- agree the structure of the handbook
- research each topic
- write each topic in Markdown
- add diagrams or visuals where useful
- review pull requests and improve content
- add the contributions table
- write the retrospective
- carry out a final team review before submission

## Workflow
- Each topic must have at least two contributors.
- All important work must be committed to GitHub.
- Work should be done on feature branches and not directly on `main`.
- Each feature branch should be merged through a pull request.
- Pull requests should be reviewed by other group members before merging.
- The handbook should be clear, practical, and easy to skim read.

## Rough Timeline and Communication
- Each day: check WhatsApp for updates, questions, and pull request review messages.
- Every second day: have a short Teams call to discuss progress, problems, and next steps.
- Start of project: finish the project plan and agree responsibilities.
- Next stage: research the three topics and collect useful sources.
- After research: write the sections in Markdown and add diagrams or visuals where useful.
- Final stage: review pull requests, complete the contributions table and retrospective, and do a final proofreading check before submission.
